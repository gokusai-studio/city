# HAKATA RISING - 実装状況 v0.1

## v0.2 追加分
- マップ上の市民ドットをタップ →「その市民の1日」詳細画面へ遷移
  - 名前・年齢・職業・暮らしぶり（短いストーリー）を自動生成して表示
  - 直近の行動履歴（自宅/通勤/勤務/余暇/帰宅の時刻付きタイムライン）を表示
- ストーリー文面は性的表現・暴力・薬物を含まない範囲で、借金・離婚・片思い・
  夜の街での息抜きなど「少し大人向け」の人間ドラマに留めています
  （Google Playコンテンツレーティングの通常審査を想定）
- **注意**：Play Console申請時の「コンテンツレーティング questionnaire」では、
  「ほのめかし程度の性的表現/夜の生活描写あり」に該当する設問に正直に回答してください。
  今の内容であれば多くの場合「Teen(12+)」相当に収まる想定ですが、最終判定はGoogle側の
  審査によります。露骨な表現へエスカレートすると審査落ち・年齢制限強化のリスクが
  上がるため、今後ストーリーを追加する際も同じトーン（婉曲表現・示唆どまり）を維持してください。

## v0.3 追加分（セーブ/資金/ミッション/広告/課金/公開準備）
- **セーブ/ロード**：Hiveでローカル保存。起動時に自動復元し、放置していた時間分の
  経済進行も簡易シミュレーションで反映（オフライン進行）
- **オートセーブ**：20秒ごとに自動保存 ＋ アプリがバックグラウンドに回った瞬間に即時保存
- **建設コスト**：道路¥100/住宅¥500/商業¥800/公園¥300を資金から消費。不足時はSnackBarで通知
- **ミッション・実績画面**：チュートリアル5項目＋実績1項目（市民詳細を見る）を進捗バー付きで表示
- **AdMobバナー広告**：画面下部に表示（現在はGoogle公式テストID。本番申請前に要差し替え）
- **アプリ内課金の基本配線**：広告非表示(非消費型)・資金パック(消費型)の購入導線を設定画面に実装
- **設定画面**：課金導線・セーブリセット・プライバシーポリシー導線
- **公開準備一式**（`store_assets/`, `RELEASE_GUIDE.md`）：
  - ストア掲載文（タイトル/説明文/コンテンツレーティング注意事項）
  - プライバシーポリシー下書き
  - アプリアイコン(512x512)・フィーチャーグラフィック(1024x500)のプレースホルダー画像
  - ビルド〜署名〜Play Console申請までの完全手順書

### ⚠️ あなたご自身で行う必要がある作業（コードでは代行不可）
1. `flutter create`でandroid/iosフォルダを生成し、本プロジェクトのlib/を統合（RELEASE_GUIDE.md 手順1）
2. AdMob/Play Consoleのアカウント作成、広告ユニットID・課金商品IDの発行と設定
3. 署名鍵(keystore)の生成・厳重保管
4. プライバシーポリシーの公開URLへの設置
5. Play Consoleでのストア情報入力・審査提出

詳細は `RELEASE_GUIDE.md` を参照してください。

## 今回実装した内容
- 博多区モチーフの初期マップ生成（那珂川・南北幹線・博多駅ランドマーク・鉄道）
- **仮想市民のリアルタイム生活シミュレーション**（新機能）
  - 統計人口とは別に、最大120体の「代表市民」が道路網をA*経路探索で移動
  - ゲーム内24時間サイクルで 自宅→職場→（時々余暇）→帰宅 を繰り返す
  - 昼夜サイクル、通勤/帰宅/余暇で色分けされたドット表示
- タイルタップでの道路/住宅/商業/公園の配置（片手操作の下部ツールバー）
- Riverpodによる状態管理、Tickerによる60FPS想定のゲームループ
- Material3テーマ（山笠の朱色ベース）
